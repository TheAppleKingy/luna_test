# Тестовое задание для МКК Луна на позицию python-разработчика

## Описание

В рамках тестового задания был реализован сервис стаб-обработки платежей. Реализованы два http-эндпоинта для создания события обработки платежа и получения данных о платеже по id, реализованом как uuid7. При отправке данных для обработки платежа генерируется событие, в последвтсвие обрабатываемое консьюмером. Для реализации фонофой обработки был использован брокер сообщений. Для простоты консьюмер, который принимает сообщения и вызывает обработчик платежа, запущен в основном потоке приложения. Для гарантии доставки сообщений был имплементирован паттерн transactional outbox. Так как логика приложения не сложная и консьюмер запущен в рамках основного потока, полезной нагрузкой в таблице outbox является id созданного платежа, который отправляется в теле сообщения брокеру. Обработчик достает по нему платеж из бд и происходит логика обработки в соответствии с тех. заданием. Также в соответствии с тех. заданием реализована retry и Dead letter политики, идемпотентность обработки платежей.

## API Эндпоинты
- POST /api/v1/payments - создание платежа. Принимает данные в виде,соответствующем тех. заданию. В соответствии в тех. заданием ожидает заголовок `Idempotency-Key`.
- GET /api/v1/payments/{payment_id} - получение данных о платеже. Возвращает все данные о платеже.

Каждый эндпоинт ожидает заголовок `X-API-Key`, использующийся для проверки доступа клиента к сервису. Ключ статический, поэтому реализована проверка устойчивая к timing analysis атакам.

## Запуск
Для простоты в репозиторий был запушен `.example.env` с переменными окружения, требуемыми приложением. Можно их изменять.
Для того, чтобы запустить приложение выполните в папке проекта:
```
docker compose up --build
```
После первого запуска примените миграции:
```
docker compose run payment_handler alembic upgrade head
```


## Тесты
Для запуска тестов выполните в папке проекта:
```
pytest -v tests/
```
Линтинг:
```
ruff check app/
```
Тайпчекинг:
```
mypy app/
```
